Automatic detection of atrial fibrillation episodes¶

    by Mai Nguyen

Introduction¶

Atrial fibrillation (AF) is a irregular heartbeat (arrhythmia) that can lead to blood clots, stroke, heart failure and other heart-related complications.

Manual detection of AF in electrocardiogram (ECG) record is time-consuming, especially in the case of large datasets consisting of 24-hour ECGs. When automating the detection, the physician can be deprived of work and research can be accelerated. Also, such an algorithm may result in the direct detection of AF during ECG monitoring, thereby creating the possibility for a fast treatment of AF.

Project Objective¶

The objective is to determine: How can machine learning algorithms and data science techniques be used to automatically detect AF? which is also the research question set for this assignment.

Data¶

The dataset we use for this study comes from Erasmus Medical Centre in Rotterdam of the department electrophysiology. There are 150,000 samples in the data, with each standing for a period of 30 seconds for a single patient. Every sample is represented by a normalized histogram of R-R intervals, and consists of 30 attributes. The label for each patient is binary, where 1 represents the existence of AF and 0 otherwise. The label distribution is not uniform, however, and only 36,537 samples (approximately 24.35%) belong to the positive class.

Methodology¶

Because our target variable, control, is binary, we recognize this problem as a classification task. Accordingly, this section presents our proposed evaluation framework to evaluate multiple classifiers on a supervised problem. For this study, we compare six different classifiers: Support Vector Machine (SVM), Multi-layer Perceptron (MLP), Random Forest, Decision Tree, Logistic Regression and Gaussian Nave Bayes.

The main flow consists of the following components:

  1. Data preparation: This step splits the original dataset into a training set for model training and optimization, and a test set for model evaluation.
  2. Data preprocessing: This step performs certain processing and transformations on the training set. Note that we do not touch the test set here, as we want the test data to be as representative of the real-world data as possible.
  3. Model optimization: Each classifier is trained and optimized using the training data. The best models found are returned for evaluation.
  4. Model evaluation: Evaluation statistics are calculated and collected by assessing the classifiers performance on the test set data.
  5. Bootstrap: The flow consisting of the previous 4 phases is repeated 1000 times, each with a different train-test split in the data preparation step. Bootstrap provides us with a means to measure confidence intervals of different evaluation metric in an empirical approach.

Main experiment function¶

The main function to carry out experiments. It also performs hyper-parameter optimization.

In [7]:
def experiment(X, y, classifier_list, param_space_list, metric_list, save_file, saving, n_tries):
    metrics = {get_classifier_name(k): {} for k in classifier_list}
    
    # We run the experiments n_tries times for bootstrapping
    for i in range(n_tries):
        print('Starting run number ' + str(i+1))
        
        # For each run, we split the dataset with a different random seed
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=i)
        
        # Iterate through each of the six classifiers
        for c_index, classifier in enumerate(classifier_list):
            classifier_name = get_classifier_name(classifier)
            
            # We wrap each classifier with sklearn's RandomizedSearchCV class to perform hyper-parameter optimization
            # for us. F1-score is used as the parameter selection measure.
            if len(param_space_list[c_index]):
                classifier_optimized = RandomizedSearchCV(classifier, param_distributions=param_space_list[c_index],
                                                          n_iter=20, n_jobs=4, cv=5, verbose=0, scoring='f1')
            else:
                classifier_optimized = classifier
            
            # LinearSVC gets a special treatment because it doesn't produce probability outputs
            # We wrap it with CalibratedClassifierCV class
            if classifier_name == 'LinearSVC':
                classifier_optimized = CalibratedClassifierCV(classifier_optimized)
            
            # Train the classifier and obtain performance metrics
            model = classifier_optimized.fit(X_train, y_train)
            
            y_pred = model.predict(X_test)
            y_prob = model.predict_proba(X_test)
            
            # Save performance scores
            for scorer in metric_scorers:
                if scorer.__name__ not in metrics[classifier_name]:
                    metrics[classifier_name][scorer.__name__] = []
                    
                if scorer.__name__ not in ['roc_curve', 'precision_recall_curve']:
                    metrics[classifier_name][scorer.__name__].append(scorer(y_test, y_pred))
                else:
                    metrics[classifier_name][scorer.__name__].append(scorer(y_test, y_prob[:,1], pos_label=1))
            
            # Save results to a pickle file
            if saving:
                with open(save_file, 'wb') as f:
                    pickle.dump(metrics, f)
    return metrics

Conclusion¶

Main observations¶

Based on the quantitative results, we conclude our study with recommendations for the research objective of building an effective machine learning pipeline to detect AF. There are two important notes:

  1. Undersampling: We use NearMiss for undersampling and it has demonstrated significant improvement compared to oversampling with Synthetic Minority Over-sampling Technique (SMOTE). Moreover, undersampling has another advantage when it comes to training speed since the training data becomes much smaller. As a result, we use undersampling when making our classifier performance.
  2. Choice of metrics: Precision and recall are suitable for our task, and should be used to dissect and understand different classifiers. As we weight these two metrics equally, we propose to use F1 Score for classifier assessment.

Recommendations¶

As different classifiers have different advantages, we propose the most promising ones for our task according to different aspects:

  • Performance: Random Forest.

From the undersampling result in Table 4.4, with a median F1 Score of 0.975, together with 95% confidence interval of F1 Score in the range between 0.969 and 0.978, Random Forest shows the best performance. One advantage of Random Forest is that by the resulting model has feature selection incorporated, allowing for much better interpretability. One disadvantage of Random Forest is that its performance results tend to have higher variance than the rest. This is the most notable in Figure 4.2 for recall.

  • Consistency: MLP.

When it comes to consistency across different sampling settings, and different evaluation metrics, MLP is the classifier to be selected. Another characteristic of MLP is that it is has a lot of rooms for engineering, and the model can get relatively complex. Deep neural networks can even learn feature representations using their hidden layers for even more robust prediction. A trade-off for choosing MLP is the difficulty in tuning the model, and the slow training process.

  • Simplicity and interpretability: Decision Tree.

Decision Tree is our pick for being a simple to train, analyze and understand model, yet its performance is on par with that of Random Forest and MLP. The training time of Decision Tree is 4-5 times faster than MLP. The tree structure of Decision Tree allows for high interpretability, and we can explore the features used easily.
