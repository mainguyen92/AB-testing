ANALYZE WERATEGODS TWITTER DATA¶

By Mai Nguyen¶

Introduction: WeRateDogs is a Twitter account that rates people's dogs with a humorous comment about the dog. These ratings almost always have a denominator of 10. The numerators, though? Almost always greater than 10. 11/10, 12/10, 13/10, etc. Why? Because "they're good dogs Brent." WeRateDogs has over 4 million followers and has received international media coverage.

The goal of this project is to wrangle WeRateDogs Twitter data to create interesting and trustworthy analyses and visualizations.

    access_secret = ''

    # For loop to append each tweet line into a list, and create a data frame from that list:
    tweet_list = []

    with open('tweet_json.txt', 'r') as tweet_json:

        for line in tweet_json:
            try:
                tweet_line = json.loads(line)
                tweet_list.append(tweet_line)
            except:
                continue

    tweet_data = pd.DataFrame(tweet_list)
    
    with open(filename, 'wb') as f:
        pickle.dump(tweet_data, f)

Quality¶

archive table¶
    • Only original ratings (no retweets) with images are needed. Some ratings do not have images.
    • Errorneous data types of timestamp (should be datetime instead of object)
    • Columns type of in_reply_to_status_id, in_reply_to_user_id and tweet_id should be string because we don't want any operations on them
    • Invalid names i.e 'None', 'O', and names whose first letters are lowercase ('a', 'the', 'very', 'quite', etc.)
    • Null values represented as 'None' (strong) in columns 'name', 'doggo', 'floofer', 'pupper', 'puppo'.
    • Rating numerators with decimals not showing full float.
    • Some ratings (numerator and denominator) are incorrect. We observe that the denominator is different from 10 for 3 reasons: there are multiple dogs rated in a tweet, or a #.# is mentioned before the real rating and is extracted, instead of the real rating (e.g. 9/11, 24/7, 3 1/2), or the tweet is not about rating dogs (tweet_id 832088576586297345, 810984652412424192).
    • The rating could be one variable, generated by dividing the numerator_rating by the denominator_rating.
    • Extra characters after '&' in the 'text' column.
images table¶
    • tweet_id should be string.
tweet_data table¶
    • There are duplicated rows.
    • id should be string. Change the name to tweet_id for joining with archive and images table.

Tidiness¶

archive table¶
    • The columns 'doggo', 'floofer', 'pupper','puppo' show one variable, representing dog stage.
    • A gender column can be created from the text columns in archive dataframe.
images table¶
    • There is no column for the most possible breed of a dog in each prediction. This column should be a part of archive table.
tweet_data table¶
    • 'tweet_data' should be a part of 'archive'.
